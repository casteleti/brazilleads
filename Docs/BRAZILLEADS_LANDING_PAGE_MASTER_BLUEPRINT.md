## 1. INFORMATION ARCHITECTURE

### Primary User Flow

```
Landing → Emotional Recognition → Value Understanding → Trust Building
→ Objection Handling → Desire Creation → Low-Friction CTA → Form Submission
→ Confirmation → Calendar Booking → Call
```

### Page Structure (Top to Bottom)

```
1. HERO (3-5 seconds of attention)
2. PROBLEM RECOGNITION (10-15 seconds)
3. POSITIONING (Understanding who you are)
4. METHODOLOGY (How you work)
5. SOCIAL PROOF (Why trust you)
6. OBJECTION HANDLING (FAQ)
7. PRICING (Transparent investment)
8. FINAL CTA (Low-friction action)
9. FOOTER (Navigation + compliance)
```

### Scroll Depth Targets

```
Hero: 0-20% (Immediate recognition)
Problem: 20-35% (Emotional hook)
Positioning: 35-50% (Brand clarity)
Methodology: 50-65% (Process confidence)
Proof: 65-80% (Social validation)
Objections: 80-90% (Risk reduction)
CTA: 90-95% (Action readiness)
Footer: 95-100% (Exit points)
```

### Navigation Model

```
No traditional navbar (keeps focus).
Logo top-left (always visible, links to top).
Sticky CTA button (bottom-right on mobile, strategic position on desktop).
WhatsApp floating button (bottom-right on mobile).
Footer navigation (for secondary paths).
```

### SECTION 1: HERO (0-20% scroll depth)

**Psychological Objective:**
Instant recognition that "these people understand me"

**Design Direction:**

- Minimalist, premium, calm
- High contrast (text readable)
- Professional photography (your face visible)
- Breathing room (generous white space)

**Layout (Desktop):**

```
Left side (60% width):
├─ Logo (small, top-left) + minimal nav
├─ Main Headline (large, bold, 48-56px)
├─ Subheadline (medium, 24-28px, secondary color)
├─ 2-3 line descriptive text (18px, supporting)
└─ CTA Button (prominent, high contrast)

Right side (40% width):
├─ Hero Image
│  ├─ Option A: Your professional photo (looking approachable)
│  ├─ Option B: Dashboard screenshot (showing growth)
│  └─ Option C: Abstract growth visualization
└─ Subtle shadow/depth
```

**Layout (Mobile):**

```
Full width stacked:
├─ Logo (smaller)
├─ Headline (36-40px, adjusted for mobile)
├─ Subheadline (20-24px)
├─ Description (16px)
├─ CTA Button (full-width, large touch target 48px+)
├─ Hero Image (full-width, optimized)
└─ Spacing: 24px top/bottom, 16px sides
```

**Copywriting Strategy - Hero:**

Headline Requirements:

- Speaks to THE pain (not solution yet)
- Specific to their situation
- Creates curiosity or recognition
- 6-12 words maximum

Subheadline Requirements:

- Clarifies what you offer
- Adds credibility signal
- Reduces initial uncertainty
- 8-15 words maximum

Description (Supporting Text):

- 2-3 sentences maximum
- Answers: "Why should I keep reading?"
- Emotional + Factual

CTA Button:

- Action-oriented
- Not salesy
- Creates low-risk perception
- Examples: "Schedule Strategy Call" / "Get Free Audit"

**UX Rationale:**

- Immediate visual hierarchy guides eyes to headline
- Photo/image adds humanity (not generic stock)
- White space reduces cognitive load
- High contrast ensures readability
- Large CTA button (but not aggressive) shows action path
- Mobile-optimized hierarchy prevents text wrapping issues

**Visual Hierarchy:**

```
Logo (5pt) → Headline (50pt) → Subheadline (25pt)
→ Description (18pt) → CTA (button, 16pt text)
```

**Color Strategy:**

- Background: White or very light gray (#FAFAFA)
- Headline: Deep blue or brand color (#1e3a8a)
- Subheadline: Secondary color (green #059669)
- CTA Button: Contrasting color (gold #f59e0b or bright green)
- Text: Dark gray (#1f2937), not pure black

**Spacing:**

- Top padding: 80px (desktop), 40px (mobile)
- Bottom padding: 80px (desktop), 60px (mobile)
- Left/right padding: 60px (desktop), 20px (mobile)
- Line spacing: 1.6 (comfortable reading)
- Letter spacing: 0 (default, no artificial spacing)

❌ Exaggerated claims ("Made $100k in 30 days")

❌ Vague results ("Life-changing!")

**Objections to Address:**

**Financial Objections:**

1. "How much does it cost?"
   → Transparent pricing (show on page)
   → ROI explanation (you make back investment in X months)
   → Flexible options (audit vs. monthly vs. combo)

2. "What's your refund/guarantee policy?"
   → 60-day guarantee explanation
   → What "success" looks like
   → How disputes are handled

3. "Why is your price higher than [competitor]?"
   → Quality difference explanation
   → Depth of service comparison
   → Long-term value vs. short-term cost

**Operational Objections:** 4. "How do time zones work?"
→ You work during their sleep → progress when they wake
→ Meeting times flexible
→ Communication via WhatsApp/Email (async-friendly)

5. "Do you need access to my business accounts?"
   → Yes, which ones (Google Ads, Facebook, etc.)
   → How security is handled
   → What you can/cannot do with access

6. "What's your onboarding process?"
   → Step-by-step walkthrough
   → Timeline (how long from signing to launch)
   → Support during onboarding

**Relationship Objections:** 7. "How often will we communicate?"
→ Monthly strategy calls (video)
→ Weekly email reports
→ WhatsApp for urgent items
→ Transparent communication style

8. "What if I need to cancel?"
   → Month-to-month (no long contracts)
   → 30-day notice period
   → Handoff of accounts
   → Partnership-first approach

9. "Do you speak Portuguese?"
   → Yes, all strategy/calls in Portuguese
   → English available but default is PT
   → Cultural understanding demonstrated

**Strategic Objections:** 10. "How do I know this will work for my business?"
→ Audit answers this (free/paid audit)
→ Case studies similar to their situation
→ Realistic expectations upfront
→ Start with limited budget option

11. "What if Google Ads doesn't work for my product?"
    → During audit, we determine if viable
    → Alternative strategies discussed
    → We're honest if not a fit

12. "How long until I see results?"
    → 15 days: noticeable changes
    → 30 days: clear results
    → 60 days: significant transformation
    → But timeline depends on situation

**Trust Objections:** 13. "Why should I trust a Brazilian agency over local ones?"
→ Cultural understanding (bilingual, bicultural)
→ Cost efficiency (60-70% less than local)
→ 24/7 coverage (work while you sleep)
→ Understanding of immigrant entrepreneur challenges

14. "What's your success rate?"
    → % of clients see improvement in 60 days (validation required)
    → Average improvement metrics (validation required)
    → Honest about what "success" means
    → Selection: We may decline non-fit clients

**UX Rationale:**

- FAQ reduces call booking anxiety
- Accordion keeps page scannable
- Honest answers build trust
- Addressing objections prevents abandonment
- Expandable format doesn't overwhelm
- Positioned before CTA (removes friction)

**Accordion Interaction:**

## 9. ACCESSIBILITY FRAMEWORK

### WCAG 2.1 AA Compliance (Minimum)

**Color Contrast:**

- Text vs. background: 4.5:1 ratio minimum
- Large text (18pt+): 3:1 ratio minimum
- Test all color combinations (design + interaction states)

**Typography:**

- Minimum font size: 14px (body text)
- Line height: 1.5+ (readability)
- Letter spacing: Normal (no artificial spacing issues)
- Font families: Sans-serif primary (readable on screen)

**Interactive Elements:**

- Focus states: Visible (blue outline, 2px minimum)
- Hover states: Distinct visual change
- Button sizes: 44x44px minimum touch target
- Skip links: Navigate to main content

**Semantic HTML:**

- Proper heading hierarchy (H1 → H2 → H3, no skips)
- Form labels: Explicitly connected to inputs
- Image alt text: Descriptive (not "image" or "photo")
- Link text: Meaningful (not "click here")

**Motion & Animation:**

- Animations: Smooth (no strobe, no rapid flashing)
- Respect `prefers-reduced-motion` (CSS media query)
- Auto-play: No audio/video auto-play
- Transitions: 300-500ms (perceptible but not slow)

**Forms:**

- Required fields: Clearly marked
- Error messages: Clear text, not just color
- Success states: Announced to screen readers
- Validation: Real-time, helpful (not punishing)

**Screen Reader:**

- Landmark regions: `<header>`, `<main>`, `<footer>`
- ARIA labels: For icon-only buttons
- Live regions: For dynamic updates (form success)
- Skip navigation: "Skip to main content"
